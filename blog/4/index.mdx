# 협업을 안해본 부서에 협업 체계 도입하기

<br />

시니어가 없는 신생 개발부서에 신입으로 들어와 이리 저리 구르며 도입한 내용을 정리합니다.

프로세스 각 단계마다 해당 기술을 도입한 이유와 적용 내용을 타임라인 순으로 정리하겠습니다.

<br />
<br />

## 도입 전 부서의 문제점

입사 전 기대와는 다르게 IT와 전혀 관계가 없던 파트내 신설된 개발 조직에 들어오게 되면서

상상과는 _많이많이_ 다른 개발 문화와 마주하였습니다.

문화 뿐만이 아니라 환경적으로도 최근 트렌드와는 거리가 멀었습니다.

기존 상황을 간략히 정리해보면,

<br />

1. 코드 공유 안됨

<br />

2. 배포 전 테스트 과정 없음

<br />

3. 개발과 운영 역할 분리 안됨

<br />

4. 개발 PM / 기획자 없음

<br />

5. 관리 계층 개발 지식 없음

<br />

지금보니까 '오 대체 어떤 곳에서 살아온거지' 하는 생각이 드는군요.
